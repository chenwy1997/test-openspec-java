# /prepare-review — PR 前变更审计摘要

## 触发方式

用户输入 `/prepare-review` 时执行本技能。

## 功能说明

在代码实现完成（`/opsx:apply` 执行后）、提交 PR 前，生成一份清晰的变更摘要，方便人工 review。

**注意**：本技能只做"整理和描述"，不做对错判断。判断由 `@reviewer`、`/spring-architecture-review`、`/sql-risk-review` 分别负责。

## 执行步骤

1. 读取当前活跃 change 目录（`openspec/changes/<change名>/`）下的 `proposal.md`、`design.md`、`tasks.md`
2. 执行 `git diff HEAD` 获取实际代码变更
3. 执行 `git status` 确认变更文件列表
4. 综合以上信息，生成结构化的 PR 摘要

## 输出格式

```markdown
## PR 变更摘要 — [Change 名称]

### 变更背景
[来自 proposal.md 的需求描述，1-3 句话]

### 影响范围

**新增文件**：
- `路径/文件名` — 说明

**修改文件**：
- `路径/文件名` — 改了什么

**删除文件**：
- `路径/文件名` — 为什么删除

### 核心逻辑变更说明
[结合 design.md，用人话描述最重要的业务逻辑变化]

### 接口变更（如有）
[新增/修改的 API 端点，包括路径、参数、返回值]

### 数据库变更（如有）
[新增/修改的表结构、索引、SQL]

### 测试覆盖说明
- ✅ 已覆盖：[场景列表]
- ⏭️ 暂未覆盖：[场景] — [原因]

### 注意事项（给 reviewer 的提示）
[有哪些地方需要特别关注，或有哪些业务背景知识有助于理解]
```
