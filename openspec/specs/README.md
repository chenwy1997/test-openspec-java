# specs/ — 当前系统规范目录

## 这个目录是什么

`specs/` 记录的是**当前系统是怎么工作的**，是系统行为的权威说明。

它不是设计文档（怎么实现），也不是需求文档（想做什么）。
它描述的是**已经实现并生效的功能规范**。

## 如何维护

**不要手动创建这里的文件**。

正确流程：
1. `/opsx:propose` 创建变更时，会在 `changes/<name>/specs/` 下生成 delta spec（增量规范）
2. `/opsx:archive` 归档时，会将 delta spec 合并到这里的主规范中

随着每次变更归档，`specs/` 会逐渐积累成完整的系统规范库。

## 目录组织方式

按**能力域（capability）**组织，每个能力域一个目录：

```
specs/
├── user/           ← 用户模块
│   └── spec.md
├── auth/           ← 认证授权
│   └── spec.md
├── order/          ← 订单模块
│   └── spec.md
└── product/        ← 商品模块
    └── spec.md
```

## spec.md 的内容格式

每个 spec.md 描述该能力域的行为规范：
- 支持哪些操作
- 业务规则是什么
- 边界条件如何处理
- 与其他模块的交互关系

**不包含**：具体实现代码、数据库 DDL、接口路径（那些在 design.md 里）
